# Функциональные требования
## Командный проект «Цветочная оранжерея» (вариант 8)

### Роли пользователей

| Роль | Описание |
|------|----------|
| **admin** | Полный доступ ко всем функциям системы. |
| **manager** | Управление растениями, композициями, заказами. Не может управлять пользователями. |
| **customer** | Только просмотр (растения, композиции, заказы, статистика). |

---

### 1. Аутентификация и авторизация
- **FR1.1** Система должна запрашивать логин и пароль при входе.
- **FR1.2** Система должна проверять логин и пароль по базе данных.
- **FR1.3** Система должна предоставлять доступ в зависимости от роли пользователя.
- **FR1.4** После трёх неудачных попыток входа доступ должен блокироваться.

---

### 2. Управление растениями
- **FR2.1** Просмотр списка всех растений (ID, название, сорт, цена, путь к фото).
- **FR2.2** Добавление нового растения (название, сорт, цена).
- **FR2.3** Редактирование данных растения.
- **FR2.4** Удаление растения (с каскадным удалением из композиций).

---

### 3. Управление композициями
- **FR3.1** Просмотр списка всех композиций.
- **FR3.2** Добавление новой композиции (название).
- **FR3.3** Редактирование названия композиции.
- **FR3.4** Удаление композиции (с каскадным удалением из заказов).
- **FR3.5** Добавление растения в композицию с указанием количества.
- **FR3.6** Просмотр состава композиции (растения, количество).
- **FR3.7** Удаление растения из композиции.
- **FR3.8** Расчёт стоимости композиции.

---

### 4. Управление заказами
- **FR4.1** Просмотр списка всех заказов.
- **FR4.2** Добавление нового заказа (клиент, композиция, количество, дата, статус).
- **FR4.3** Редактирование статуса заказа.
- **FR4.4** Удаление заказа.
- **FR4.5** Завершение заказа (установка даты выполнения и статуса `completed`).
- **FR4.6** Поиск заказов по статусу.
- **FR4.7** Поиск заказов по клиенту (по шаблону).
- **FR4.8** Поиск заказов за период.

---

### 5. Статистика и отчёты
- **FR5.1** Общая выручка (сумма стоимостей выполненных заказов).
- **FR5.2** Самые популярные композиции (по количеству заказов).
- **FR5.3** Количество заказов по статусам.

---

### 6. Хранение изображений
- **FR6.1** Система должна хранить пути к фотографиям растений в базе данных.
- **FR6.2** При выводе списка растений должны отображаться пути к фото.
- **FR6.3** Система должна обеспечивать возможность добавления фото к новому растению.
