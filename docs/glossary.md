# Глоссарий
## Командный проект «Цветочная оранжерея» (вариант 8)

| Термин | Описание |
|--------|----------|
| **Растение (Plant)** | Цветок, выращиваемый в оранжерее. Имеет название, сорт, цену за единицу и путь к фотографии. |
| **Композиция (Composition)** | Набор растений, объединённых в единое целое. Имеет название и состав (список растений с количеством). |
| **Заказ (Order)** | Запрос клиента на изготовление определённого количества композиций. Содержит ФИО клиента, название композиции, количество, дату заказа, дату выполнения и статус. |
| **Срочный заказ (Urgent Order)** | Заказ, который должен быть выполнен в течение суток. Дополнительная плата — 25% от стоимости. |
| **Обычный заказ (Regular Order)** | Заказ, выполняемый в течение нескольких дней. |
| **Статус заказа (Order Status)** | Состояние заказа: `pending` (ожидает), `completed` (выполнен), `urgent` (срочный). |
| **Выручка (Revenue)** | Общая сумма денег, полученная от выполненных заказов. Рассчитывается как сумма стоимостей всех композиций в заказе. |
| **Аутентификация (Authentication)** | Процесс проверки логина и пароля пользователя. |
| **Авторизация (Authorization)** | Определение прав доступа пользователя (admin, manager, customer). |
| **CRUD** | Акроним для операций: Create (создание), Read (чтение), Update (обновление), Delete (удаление). |
| **SQLite** | Лёгкая реляционная база данных, используемая для хранения информации о растениях, композициях, заказах и пользователях. |
| **Google Test** | Фреймворк для написания и запуска unit-тестов на C++. |
| **GitHub Actions** | Система непрерывной интеграции (CI), автоматически собирающая проект и запускающая тесты при каждом push. |
| **Makefile** | Файл сценария для утилиты make, описывающий процесс сборки проекта. |
| **Модель КИС** | Модель организации кода: Клиент-Интерфейс-Сервер. |
